# Настройка SMTP для «Кино на крыше»

Приложение отправляет два типа писем: **код подтверждения email** (6 цифр, живёт 10 минут) и **ссылка для сброса пароля** (живёт 1 час). Если в `backend/.env` не указан SMTP-сервер, письма выводятся в **консоль backend** с пометкой `[DEV-EMAIL]` — удобно для разработки.

> **TL;DR.** Самый простой стабильный вариант для российских проектов — **Яндекс 360 для бизнеса** (если есть свой домен) или **UniSender / SendPulse** (если домена нет). Для продакшена с высокой нагрузкой — **Mailgun** или **Postmark**.

## Где это настраивается

Все настройки — в `backend/.env`:

```env
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=587
SMTP_USER=noreply@yourdomain.ru
SMTP_PASSWORD=пароль_приложения_или_API_ключ
SMTP_FROM=noreply@yourdomain.ru
SMTP_USE_TLS=true
APP_BASE_URL=https://your.deployed.host
```

После изменения `.env` **перезапустите backend** — настройки читаются только при старте.

`APP_BASE_URL` используется при формировании ссылки в письме сброса пароля (например, `${APP_BASE_URL}/reset-password?token=...`). Если запускаете локально для теста с телефона — поставьте `http://192.168.x.x:5180` (свой LAN IP).

## Российские провайдеры

### Яндекс 360 (smtp.yandex.ru)

**Лучший выбор, если у вас есть свой домен** и небольшой объём отправки. Стабильно работает в РФ, не требует прокси.

**Бесплатный тариф:**
- До 3 пользователей на ваш домен.
- ~500 писем в день с одного ящика.
- 30 ГБ почты.

**Платные:** «Базовый» от 249 ₽/мес за пользователя, лимит ~5000 писем/день.

**Как настроить:**
1. Зарегистрируйте свой домен в [Yandex 360 для бизнеса](https://360.yandex.ru/business/).
2. Подтвердите домен (TXT-запись в DNS).
3. Создайте ящик `noreply@yourdomain.ru` (или используйте существующий).
4. Войдите в этот ящик через web → [«Управление аккаунтом» → «Пароли приложений»](https://id.yandex.ru/security/app-passwords) → создайте пароль для «Почты».
5. Заполните `.env`:
   ```env
   SMTP_HOST=smtp.yandex.ru
   SMTP_PORT=587
   SMTP_USER=noreply@yourdomain.ru
   SMTP_PASSWORD=сгенерированный_пароль_приложения
   SMTP_FROM=noreply@yourdomain.ru
   SMTP_USE_TLS=true
   ```

**Альтернатива (без своего домена):** обычный Яндекс.Почта `@yandex.ru` — тоже работает по тем же настройкам, но письма с @yandex.ru у получателей могут чаще попадать в спам, и есть жёсткий лимит в ~150 писем в час.

### Mail.ru

**Лимит:** 200 писем/день для обычного ящика, до 30 за час, не более 30 получателей в одном письме.

**Минусы:** письма часто попадают в спам у получателей-провайдеров (Gmail в первую очередь).

**Настройка:**
1. Войдите в свой ящик `@mail.ru` → «Пароли для внешних приложений» → создайте пароль.
2. Конфиг:
   ```env
   SMTP_HOST=smtp.mail.ru
   SMTP_PORT=465                # SSL
   SMTP_USER=youname@mail.ru
   SMTP_PASSWORD=пароль_приложения
   SMTP_FROM=youname@mail.ru
   SMTP_USE_TLS=false           # для 465 используется SSL, не STARTTLS
   ```
   ⚠️ Текущая реализация backend использует STARTTLS на порту 587. Для Mail.ru это тоже работает: `SMTP_PORT=587`, `SMTP_USE_TLS=true`.

### UniSender Go

Транзакционный сервис для рассылок, **российский**, без VPN/прокси.

**Бесплатно:** 100 писем в день.  
**Платно:** от 200 ₽/мес за 1000 писем, далее по тарифной сетке.

1. Регистрация на [unisender.com](https://www.unisender.com/) → раздел «UniSender Go» → подтвердите ваш домен (DKIM + SPF записи в DNS).
2. Создайте API-ключ в разделе «Настройки» → «Доступ через SMTP».
3. Конфиг:
   ```env
   SMTP_HOST=smtp.go1.unisender.ru
   SMTP_PORT=587
   SMTP_USER=ваш_login_из_unisender
   SMTP_PASSWORD=ваш_api_key
   SMTP_FROM=noreply@yourdomain.ru   # обязательно совпадает с подтверждённым доменом
   SMTP_USE_TLS=true
   ```

### SendPulse

Гибрид CRM + рассылок. Российская юрисдикция, поддержка SMTP.

**Бесплатно:** 12 000 писем/мес, лимит 200/час.  
**Платно:** от ~$8/мес за более высокие лимиты.

1. [sendpulse.ru](https://sendpulse.ru/) → SMTP → подтвердите домен.
2. Получите SMTP-логин и пароль из настроек.
3. Конфиг:
   ```env
   SMTP_HOST=smtp-pulse.com
   SMTP_PORT=465
   SMTP_USER=ваш_login
   SMTP_PASSWORD=ваш_smtp_password
   SMTP_FROM=noreply@yourdomain.ru
   SMTP_USE_TLS=true
   ```

### Notisend / Mailopost

Два менее известных российских сервиса, специализирующихся на транзакционных письмах.

- **Notisend** (notisend.ru): от 200 ₽/мес, до 10 000 писем в день.
- **Mailopost** (mailopost.ru): бесплатно до 1500 писем/мес.

Настройка типовая: SMTP_HOST из их документации, SMTP_USER/SMTP_PASSWORD — учётка/API-ключ, обязательное подтверждение домена через DNS-записи.

## Зарубежные провайдеры

> **Важно:** ряд зарубежных сервисов (Mailchimp, SendGrid, частично Gmail) могут блокировать или замедлять отправку из РФ. Если backend хостится в РФ — берите российский провайдер или прокси к зарубежному API.

### Gmail

Подходит для разработки и совсем маленьких проектов.

**Лимит:** 500 писем в сутки для бесплатного `@gmail.com`, до 2000 для Google Workspace.

1. Включите [двухфакторную авторизацию](https://myaccount.google.com/security) в Google-аккаунте.
2. Создайте [пароль приложения](https://myaccount.google.com/apppasswords) для «Mail».
3. Конфиг:
   ```env
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_USER=youname@gmail.com
   SMTP_PASSWORD=шестнадцатисимвольный_пароль_приложения
   SMTP_FROM=youname@gmail.com
   SMTP_USE_TLS=true
   ```

⚠️ Письма с `@gmail.com` чаще приземляются в спам у российских получателей.

### SendGrid

Профессиональный SMTP/API-сервис, ныне в составе Twilio. Лёгкая интеграция, но **из РФ часто требует VPN**.

**Бесплатно:** 100 писем/день навсегда.  
**Платно:** от $20/мес за 50 000 писем.

1. Регистрация на [sendgrid.com](https://sendgrid.com).
2. Подтвердите Single Sender либо домен.
3. Создайте API-ключ с правом «Mail Send».
4. Конфиг:
   ```env
   SMTP_HOST=smtp.sendgrid.net
   SMTP_PORT=587
   SMTP_USER=apikey
   SMTP_PASSWORD=ваш_api_ключ
   SMTP_FROM=noreply@yourdomain.ru
   SMTP_USE_TLS=true
   ```

### Mailgun

Любимец стартапов, хорошие лимиты, документация на 5+.

**Лимиты:** первые 3 месяца — 5000 писем/мес бесплатно, далее $35/мес за 50 000.  
**Из РФ:** работает напрямую без VPN, но может потребоваться прокси на отправляющей стороне в зависимости от региона.

1. Регистрация на [mailgun.com](https://www.mailgun.com).
2. Подтвердите домен (4 DNS-записи).
3. В разделе «Sending → Domain Settings → SMTP credentials» возьмите логин/пароль.
4. Конфиг:
   ```env
   SMTP_HOST=smtp.eu.mailgun.org      # или smtp.mailgun.org для US-региона
   SMTP_PORT=587
   SMTP_USER=postmaster@mg.yourdomain.ru
   SMTP_PASSWORD=сгенерированный_пароль
   SMTP_FROM=noreply@yourdomain.ru
   SMTP_USE_TLS=true
   ```

### Postmark

Премиум-сервис только для **транзакционных** писем (логины, регистрации, чеки). Не для рассылок.

**Бесплатно:** 100 писем/мес.  
**Платно:** от $15/мес за 10 000 писем. Лучшая доставляемость в индустрии.

1. Регистрация на [postmarkapp.com](https://postmarkapp.com).
2. Подтвердите домен.
3. Создайте Server → возьмите SMTP-токен.
4. Конфиг:
   ```env
   SMTP_HOST=smtp.postmarkapp.com
   SMTP_PORT=587
   SMTP_USER=ваш_серверный_токен
   SMTP_PASSWORD=ваш_серверный_токен   # юзер и пароль одинаковые
   SMTP_FROM=noreply@yourdomain.ru
   SMTP_USE_TLS=true
   ```

### Brevo (бывший Sendinblue)

**Бесплатно:** 300 писем в сутки (≈9000 в месяц).  
**Платно:** от $9/мес за 20 000 писем.

1. Регистрация на [brevo.com](https://www.brevo.com).
2. «SMTP & API» → создайте SMTP-ключ.
3. Конфиг:
   ```env
   SMTP_HOST=smtp-relay.brevo.com
   SMTP_PORT=587
   SMTP_USER=ваш_email_из_brevo
   SMTP_PASSWORD=ваш_SMTP_ключ
   SMTP_FROM=noreply@yourdomain.ru
   SMTP_USE_TLS=true
   ```

### Amazon SES

Самый дешёвый при большом объёме. Подходит, если уже используете AWS.

**Бесплатно:** 62 000 писем/мес, если backend хостится на EC2.  
**Платно:** $0.10 за 1000 писем — почти даром.

⚠️ Sandbox-режим первые недели — можно отправлять только на подтверждённые адреса. Нужно подать запрос на выход из песочницы.

```env
SMTP_HOST=email-smtp.eu-central-1.amazonaws.com   # или другой регион
SMTP_PORT=587
SMTP_USER=AKIA...                                 # SMTP credentials (не путать с AWS API key)
SMTP_PASSWORD=сгенерированный_smtp_password
SMTP_FROM=noreply@yourdomain.ru
SMTP_USE_TLS=true
```

## Сравнение лимитов

| Провайдер | Бесплатно | Платно (старт) | Из РФ | Свой домен нужен |
|-----------|-----------|----------------|-------|------------------|
| **Яндекс 360** | 500/день, 3 юзера | от 249 ₽/мес/юзер | ✅ напрямую | да |
| **Mail.ru** | 200/день | — | ✅ | нет |
| **UniSender Go** | 100/день | от 200 ₽/мес | ✅ | да |
| **SendPulse** | 12 000/мес | от $8/мес | ✅ | да |
| **Brevo** | 300/день | от $9/мес | ✅ обычно | да |
| **Gmail** | 500/день | 2000/день (Workspace) | ✅ | нет |
| **Mailgun** | 5000 первые 3 мес | $35/мес за 50k | ✅ обычно | да |
| **SendGrid** | 100/день | $20/мес за 50k | ⚠️ возможны блоки | да |
| **Postmark** | 100/мес | $15/мес за 10k | ⚠️ | да |
| **Amazon SES** | 62k/мес (с EC2) | $0.10 за 1k | ✅ | да |

**Рекомендации для «Кино на крыше»:**

- **Только что запускаетесь, тестируете** → начните без SMTP. Письма из консоли backend — этого достаточно для разработки и пилотных тестов.
- **Российская аудитория, MVP** → Яндекс 360 (если есть домен) либо UniSender Go (100/день бесплатно).
- **Российская аудитория, рабочий проект** → SendPulse, Notisend или Яндекс 360 (платный).
- **Международный проект** → Mailgun или Brevo, для критичных транзакционных писем — Postmark.
- **Тысячи писем в день и больше** → Amazon SES (если уже на AWS) или Mailgun.

## Что делать, если письма не доходят

### Письмо не пришло
1. Проверьте папку **«Спам»** у получателя.
2. Посмотрите консоль backend — есть ли ошибка SMTP? В коде есть `log.exception("SMTP send failed")` — она появится в выводе uvicorn.
3. Убедитесь, что `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD` указаны верно. Не используйте обычный пароль от ящика — нужен **пароль приложения** (для Yandex/Gmail/Mail.ru) или **API-ключ** (для SendGrid/Mailgun/Postmark).
4. Проверьте, что `SMTP_FROM` совпадает с доменом, который вы подтвердили у провайдера. Иначе отправка отклоняется.

### Письмо в спаме
1. Настройте **SPF**, **DKIM**, **DMARC** записи в DNS (см. ниже).
2. Используйте свой домен в `SMTP_FROM`, не публичную почту типа `@gmail.com` или `@yandex.ru`.
3. Не пишите «спамные» темы в коде: уберите слова «срочно», «бесплатно», много восклицательных знаков из текста писем.
4. Прогрейте репутацию IP/домена постепенно — не шлите 1000 писем в первый же час.

### Ошибка `535 Authentication failed` или `530 Authentication required`
Неправильный логин/пароль. Для Yandex/Gmail/Mail.ru — проверьте, что используете пароль приложения, а не обычный пароль.

### Ошибка `550 5.7.1 Sender address rejected`
`SMTP_FROM` не совпадает с подтверждённым доменом провайдера. Поменяйте на правильный.

### Ошибка `421 Service not available`
Превышен дневной/часовой лимит. Подождите или смените провайдера.

### Ошибка `Connection refused` / `timed out`
Брандмауэр или провайдер блокирует исходящие SMTP. Попробуйте порт 465 + SSL (вместо 587 + STARTTLS) либо используйте API-провайдера через HTTPS.

## Защита от спам-папки: SPF, DKIM, DMARC

Эти три DNS-записи нужны, чтобы получатели (особенно Gmail) доверяли вашим письмам.

### SPF (Sender Policy Framework)

TXT-запись на домене: указывает, какие сервера имеют право отправлять почту от вашего имени.

```
v=spf1 include:_spf.yandex.net ~all
```
(для Yandex; у каждого провайдера свой `include:`).

### DKIM (DomainKeys Identified Mail)

Цифровая подпись писем. Провайдер выдаст готовый ключ — добавляете TXT-запись с именем вида `mail._domainkey.yourdomain.ru`.

### DMARC (Domain-based Message Authentication, Reporting & Conformance)

Политика: что делать с письмами, которые не прошли SPF/DKIM.

```
_dmarc.yourdomain.ru.  TXT  "v=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.ru"
```

После настройки всех трёх записей **подождите 24 часа** (DNS-кеш), затем проверьте на [mxtoolbox.com](https://mxtoolbox.com/) или [mail-tester.com](https://www.mail-tester.com/).

## Локальная отладка без SMTP

Оставьте `SMTP_HOST=` пустым в `.env`. Тогда:
- Письма выводятся в консоль backend с пометкой `[DEV-EMAIL]`.
- Удобно тестировать flow подтверждения email и сброса пароля без реальной отправки.

Пример вывода:
```
[DEV-EMAIL]
To: user@test.com
Subject: Подтверждение email — Кино на крыше

Код подтверждения email: 087898

Введите его на странице регистрации. Код действует 10 минут.
```

Скопируйте код из консоли и введите в форме `/verify-email` в браузере.

То же самое для сброса пароля — скопируйте ссылку из лога и вставьте в браузер.
